# 4단계: Pain Point 분석

## 단계 목표
여정지도에서 도출된 Pain Point를 체계적으로 분석하여 "무엇이 불편한가"를 명확히 하고 우선순위를 설정합니다.

---

## 입력 필요 산출물
- 3단계: 사용자 여정지도 (단계별 Pain Point, 기회영역)
- 2단계: 페르소나 (목표, 불만사항)

---

## 프롬프트

### 4-1. Pain Point 추출 프롬프트

```
AI서비스 UIUX 설계 중. 4단계 Pain Point 분석.

[여정지도]
{3단계 여정지도 결과 붙여넣기}

여정지도에서 Pain Point 추출 및 분석:

| ID | Pain Point | 발생 단계 | 원인 | 심각도(상/중/하) | 빈도(상/중/하) | 우선순위 |

심각도 × 빈도로 우선순위 산정.
각 Pain Point별로 AI로 해결 가능한지 여부 표시.
```

### 4-2. Pain Point 심화 분석 프롬프트

```
우선순위 상위 3개 Pain Point에 대해:

1. 근본 원인 분석 (5 Whys)
2. 현재 대안/우회 방법
3. AI 기반 해결 방향
4. 해결 시 기대 효과

각각 상세히 분석.
```

### 4-3. HMW 질문 도출 프롬프트

```
Pain Point를 HMW(How Might We) 질문으로 전환:

Pain Point: {Pain Point 내용}
→ HMW: "어떻게 하면 ~할 수 있을까?"

상위 5개 Pain Point에 대해 HMW 질문 작성.
AI 서비스 관점에서 실현 가능한 방향 제시.
```

```
        빈도 높음
            │
     ┌──────┼──────┐
     │  2순위 │ 1순위  │
     │ 중요  │ 긴급+중요│
심각도├──────┼──────┤심각도

     └──────┼──────┘
            │
        빈도 낮음
```
