# AI 서비스 UI/UX 설계 실습 - 마스터 지침

## 실습 개요

본 실습은 디자인씽킹 방법론을 적용하여 AI 서비스의 UI/UX를 설계하고 프로토타입까지 제작하는 과정입니다.

---

## 프로세스 구조

```
[Discover]          [Define]              [Develop]              [Deliver]
맥락 이해            경험 시각화            구조화                  설계/시각화
→ 사용자 정의        → 문제 발굴            → 해결책 도출           → 프로토타입

┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│1. 업무  │→│2. 페르  │→│3. 여정  │→│4. Pain  │→│5. 멘탈  │→│6. IA    │→│7. 기능  │→│8. UI/UX │→│9. 프로토│
│   분석  │  │   소나  │  │   지도  │  │  Point  │  │  모형   │  │   설계  │  │   설계  │  │   설계  │  │   타입  │
└─────────┘  └─────────┘  └─────────┘  └─────────┘  └─────────┘  └─────────┘  └─────────┘  └─────────┘  └─────────┘
```

---

## 핵심 원칙

**앞 단계 산출물이 뒷 단계의 품질을 결정합니다.**

| 원인 | 결과 |
|------|------|
| 페르소나 없이 | 여정지도 작성 불가 |
| Pain Point 없이 | 기능 우선순위 불가 |
| 멘탈모형 없이 | 사용자 관점 IA 설계 불가 |
| IA 없이 | 화면 설계 불가 |

## 산출물 연계 흐름

```
INPUT                                                                      OUTPUT
업무분석서 ─────→ 여정지도 ─────→ 멘탈모형 ─────→ 정보구조도 ─────→ UI/UX 설계
(목적/프로세스/현안)  (어떻게 경험?)   (어떻게 인식?)   (어떻게 구성?)     (어떤 원칙?)
      │                  │               │                │                  │
      ▼                  ▼               ▼                ▼                  ▼
  페르소나          Pain Point       개념 분류표        기능설계           프로토타입
  (누가 사용?)      (무엇이 불편?)    (사용자 언어)     (무엇을 제공?)     (시각화된 결과물)
```

## 실습 진행 방법

### 무료 생성형AI 활용 시

1. **각 단계 파일 열기** → 프롬프트 섹션 복사
2. **이전 단계 산출물 첨부** → 프롬프트와 함께 입력
3. **결과 검토 및 수정** → 산출물 섹션에 기록
4. **다음 단계로 이동**

### 대화가 길어질 경우

- 새 대화 시작
- 해당 단계 프롬프트 + 필요한 이전 산출물만 첨부
- 토큰 절약 가능

## 주의사항

- 각 단계를 건너뛰지 마십시오
- 산출물은 반드시 문서화하여 다음 단계에 전달하십시오
- AI 응답을 그대로 사용하지 말고, 검토 후 수정/보완하십시오
- 프로토타입 단계에서는 MVP 범위에 집중하십시오
