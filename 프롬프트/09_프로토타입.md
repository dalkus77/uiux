# 9단계: 프로토타입

## 단계 목표
UI/UX 설계를 기반으로 "시각화된 결과물"인 프로토타입을 제작합니다.

---

## 입력 필요 산출물
- 7단계: 기능설계 (MVP 기능, 기능 상세 스펙, 핵심 기능 워크플로우)
- 8단계: UI/UX 설계 (화면 레이아웃, AI UI 패턴, 컴포넌트 명세)

---

## 프롬프트

### 9-1. 프로토타입 코드 생성 프롬프트 (React)

```
AI서비스 프로토타입 제작. React 컴포넌트로 구현.

[화면 구조]
{8단계 화면 레이아웃 붙여넣기}

[컴포넌트 명세]
{8단계 컴포넌트 명세 붙여넣기}

[AI UI 패턴]
{8단계 AI UI 패턴 붙여넣기}

다음 화면 프로토타입 코드 작성:
- 화면: {화면명}
- 프레임워크: React + Tailwind CSS
- 상태관리: useState
- AI 기능: 목업 데이터로 시뮬레이션

컴포넌트 분리하여 작성.
```

### 9-2. 프로토타입 코드 생성 프롬프트 (HTML)

```
AI서비스 프로토타입 제작. HTML/CSS/JS 단일 파일로 구현.

[화면 구조]
{8단계 화면 레이아웃 붙여넣기}

[컴포넌트 명세]
{8단계 컴포넌트 명세 붙여넣기}

다음 화면 프로토타입 코드 작성:
- 화면: {화면명}
- 스타일: 모던한 UI, 반응형
- AI 기능: 버튼 클릭 시 목업 결과 표시

단일 HTML 파일로 작성. 외부 의존성 최소화.
```

### 9-3. 인터랙션 추가 프롬프트

```
프로토타입에 다음 인터랙션 추가:

[현재 코드]
{생성된 프로토타입 코드}

추가할 인터랙션:
1. {화면 전환 / 모달 / 애니메이션 등}
2. AI 결과 로딩 애니메이션
3. 피드백 버튼 동작
4. 폼 검증

사용자 경험 향상 중심으로 구현.
```

### 9-4. 다중 화면 연결 프롬프트

```
프로토타입 화면 간 연결:

[화면 1 코드]
{화면 1 코드}

[화면 2 코드]
{화면 2 코드}

React Router 또는 상태 기반으로 화면 전환 구현.
네비게이션 흐름:
{6단계 사용자 플로우 붙여넣기}
```
