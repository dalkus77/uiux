# 2단계: 페르소나

## 단계 목표
요구사항 정의서의 대상 사용자를 기반으로 구체적인 페르소나를 작성하여 "누가 이 서비스를 사용하는가"를 명확히 합니다.

---

## 입력 필요 산출물
- 1단계: 요구사항 정의서 (대상 사용자, 업무 프로세스, 현안)

---

## 프롬프트

### 2-1. 페르소나 생성 프롬프트

```
AI서비스 UIUX 설계 중. 2단계 페르소나 작성.

[이전 산출물: 요구사항 정의서]
{1단계 결과 중 대상 사용자, 업무 프로세스, 현안 부분 붙여넣기}

위 요구사항 기반으로 주요 사용자 페르소나 2-3개 작성:

각 페르소나별로:
- 이름/나이/직책
- 역할 및 담당 업무
- 목표 (이 서비스로 달성하려는 것)
- 불만/Pain Point (현재 겪는 어려움)
- 기술 숙련도 (상/중/하)
- 서비스 이용 맥락 (언제, 어디서, 어떤 상황에서)
- 핵심 니즈 한 문장

간결한 표 형식으로.
```

### 2-2. 페르소나 검증 프롬프트 (선택)

```
작성한 페르소나가 다음 기준을 충족하는지 검토해줘:
1. 요구사항의 대상 사용자를 대표하는가?
2. 목표와 Pain Point가 구체적인가?
3. 서비스 설계에 실질적 인사이트를 주는가?
4. 서로 다른 사용자 유형을 커버하는가?

부족한 부분 보완 제안.
```

### 2-3. 페르소나 우선순위 선정 프롬프트 (선택)

```
페르소나 선정을 위해 2x2 매트릭스로 평가해줘.

[페르소나 목록]
{작성된 페르소나 요약}

평가 축:
- X축: 기술 숙련도 (낮음 ↔ 높음)
- Y축: 서비스 필요 절박성 또는 위험 노출도 (낮음 ↔ 높음)

매트릭스에 페르소나 배치하고, 핵심 타겟 페르소나 선정 근거 제시.
```
