# 5단계: 멘탈모형 설계

## 단계 목표
사용자가 정보를 어떻게 인식하고 분류하는지 파악하여 "사용자 관점의 정보 구조"를 도출합니다. 이를 통해 시스템 중심이 아닌 사용자 중심의 IA를 설계할 수 있습니다.

---

## 입력 필요 산출물
- 2단계: 페르소나 (기술 숙련도, 배경, 목표)
- 3단계: 사용자 여정지도 (단계, 터치포인트)
- 4단계: Pain Point 분석 (HMW 질문, 해결 방향)

---

## 멘탈모형이란?

멘탈모형(Mental Model)은 **사용자가 시스템이나 정보를 어떻게 이해하고 예상하는지**에 대한 내적 표상입니다.

| 구분 | 시스템 모델 | 멘탈모형 |
|------|-----------|---------|
| 관점 | 개발자/설계자 | 사용자 |
| 분류 기준 | 기능, 기술, 데이터 구조 | 목표, 상황, 익숙한 개념 |
| 언어 | 전문 용어 | 일상 용어 |
| 예시 | "센서 데이터 분석" | "지금 안전한가?" |

---

## 프롬프트

### 5-1. 멘탈모형 다이어그램 생성 프롬프트

```
AI서비스 UIUX 설계 중. 5단계 멘탈모형 설계.

[페르소나]
{2단계 핵심 타겟 페르소나 붙여넣기}

[사용자 여정지도]
{3단계 여정지도 단계별 요약 붙여넣기}

[Pain Point 및 HMW]
{4단계 상위 Pain Point와 HMW 질문 붙여넣기}

이 페르소나 관점에서 멘탈모형 다이어그램을 작성해줘.

다음 3계층 구조의 표로 출력:

1. 과업 단계 (Task Stage): 사용자가 목표 달성을 위해 거치는 큰 단계 (4-6개)
   - 여정지도 기반으로 사용자 관점의 단계명 작성
   - 예: "상황 인식", "방법 탐색", "실제 수행", "결과 확인", "학습/개선"

2. 사용자 니즈 (User Needs): 각 과업 단계에서 사용자가 원하는 것 (단계당 3-5개)
   - 사용자 언어로 작성 (전문용어 X)
   - "~하기", "~확인하기", "~알아보기" 형태

3. 시스템 기능 (System Features): 각 사용자 니즈를 해결하는 기능 (니즈당 1-2개)
   - 구체적인 기능명으로 작성

표 형식:
| 과업 단계 | 사용자 니즈 | 시스템 기능 |
|----------|-----------|------------|
| 단계1 | 니즈1-1 | 기능1-1 |
| | 니즈1-2 | 기능1-2 |
| 단계2 | 니즈2-1 | 기능2-1 |
| ... | ... | ... |
```

### 5-2. 사용자 용어 추출 프롬프트

```
위 멘탈모형에서 사용된 사용자 용어와 시스템 용어를 대응표로 정리해줘:

| 사용자 용어 (일상어) | 시스템 용어 (전문어) | 적용 위치 |
|-------------------|-------------------|----------|

메뉴명, 버튼명, 안내문구에 사용할 수 있도록 정리.
```

### 5-3. IA 설계 가이드라인 도출 프롬프트

```
멘탈모형 다이어그램 기반으로 IA 설계 가이드라인 도출해줘:

1. 메뉴 구조 제안: 과업 단계 → 1depth 메뉴로 전환
2. 레이블링 원칙: 사용자 용어 적용 방안
3. 정보 우선순위: 어떤 정보를 먼저 보여줄 것인가
4. 네비게이션 흐름: 과업 단계 순서에 맞는 흐름

표 형식으로 정리.
```
