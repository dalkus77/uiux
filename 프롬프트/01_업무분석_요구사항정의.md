# 1단계: 업무분석 및 요구사항 정의

## 단계 목표
RFP(제안요청서) 문서를 분석하여 프로젝트의 업무 목적, 프로세스, 현안을 파악하고 요구사항을 체계적으로 정의합니다.

---

## 프롬프트

### 1-1. RFP 분석 프롬프트

```
첨부한 RFP PDF를 분석해서 다음 항목을 추출해줘:

1. 프로젝트 개요
   - 사업명
   - 목적
   - 범위

2. 업무 현황
   - 현재 프로세스
   - 현안/문제점

3. 요구사항 분류
   - 기능 요구사항 (FR)
   - 비기능 요구사항 (NFR)
   - 데이터 요구사항
   - AI/ML 관련 요구사항

4. 제약조건
   - 기술적 제약
   - 일정/예산 제약
   - 기타 제약

표 형식으로 정리. 불명확한 항목은 [확인필요]로 표시.
```

### 1-2. 요구사항 상세화 프롬프트 (선택)

```
위 분석 결과 중 기능 요구사항을 상세화해줘:

요구사항ID | 요구사항명 | 상세설명 | 우선순위(필수/권장/선택) | 관련 현안

AI 서비스 특성을 고려해서 다음 관점도 포함:
- 사용자 입력 처리 방식
- AI 응답/추천 방식
- 오류/예외 처리
- 설명가능성(XAI) 요구
```
