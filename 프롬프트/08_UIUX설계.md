# 8단계: UI/UX 설계

## 단계 목표
기능 설계를 기반으로 "어떤 원칙으로 설계할 것인가"를 정의하고 화면 구조를 설계합니다.

---

## 입력 필요 산출물
- 2단계: 페르소나 (기술 숙련도, 이용 맥락)
- 5단계: 멘탈모형 설계 (사용자 언어, 핵심 질문)
- 6단계: IA 설계 (내비게이션 구조, 사용자 플로우)
- 7단계: 기능설계 (MVP 기능, AI 기능 UI 표현, 핵심 기능 워크플로우)

---

## 프롬프트

### 8-1. UI 설계원칙 수립 프롬프트

```
AI서비스 UIUX 설계 중. 8단계 UIUX 설계원칙.

[페르소나]
{2단계 기술 숙련도, 이용 맥락 붙여넣기}

[멘탈모형]
{5단계 사용자 핵심 질문, 용어 붙여넣기}

[AI 기능]
{7단계 AI 기능 목록 붙여넣기}

다음 작성:

1. UI 설계원칙 5개
   - 원칙명
   - 설명
   - 적용 예시

2. AI 서비스 특화 UX 원칙
   - 신뢰성 구축 방안
   - 설명가능성(XAI) 표현
   - 오류/불확실성 처리
   - 사용자 제어권 보장
```

### 8-2. 색상 가이드 도출 프롬프트

```
UI 색상 가이드를 도출해줘.

[서비스 특성]
- 서비스명: {서비스명}
- 타겟 사용자: {페르소나 요약}
- 서비스 성격: {신뢰/안전/혁신/친근 등 키워드}
- 사용 환경: {실내/야외/모바일/데스크톱 등}

[색상 도출 조건]

1. 레퍼런스 기반 접근
   - 동종 업계 TOP 3 서비스의 색상 분석
   - 차별화되면서도 업계 맥락에 맞는 색상 제안
   - 참고할 디자인 트렌드: {2024-2025 트렌드 or 특정 스타일}

2. 피해야 할 색상 조합 (AI스러운 느낌)
   - 과채도 파랑+보라 그라데이션
   - 네온 컬러 과다 사용
   - 무채색+단일 포인트 컬러의 뻔한 조합
   - 모든 색상이 같은 채도/명도인 조합

3. 추구할 색상 특성
   - 주조색: 브랜드 아이덴티티를 담되, 미묘한 톤 변화가 있는 색상
   - 보조색: 주조색과 의외의 조화를 이루는 색상 (유사색 X)
   - 강조색: 절제된 사용으로 시선을 끄는 악센트
   - 의미색: 상태 표시용 (성공/경고/오류) - 접근성 고려

4. 출력 형식
   | 구분 | 색상명 | HEX | 사용 용도 | 선정 이유 |
   |------|--------|-----|----------|----------|
   | 주조색 | | | | |
   | 보조색 | | | | |
   | 강조색 | | | | |
   | 배경색 | | | | |
   | 텍스트색 | | | | |
   | 성공 | | | | |
   | 경고 | | | | |
   | 오류 | | | | |

5. 색상 조합 검증
   - 명도 대비: 텍스트 가독성 (WCAG AA 이상)
   - 색맹 고려: 적록색맹 사용자도 구분 가능한지
   - 감정 연상: 타겟 사용자에게 적절한 감정을 유발하는지
```
