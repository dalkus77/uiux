# 7단계: 기능설계

## 단계 목표
IA와 Pain Point를 기반으로 "무엇을 제공할 것인가"를 구체적으로 정의하고 기능 명세를 작성합니다.

---

## 입력 필요 산출물
- 1단계: 요구사항 정의서 (기능/비기능/AI 요구사항)
- 4단계: Pain Point 분석 (해결 방향)
- 6단계: IA 설계 (메뉴별 기능, 사용자 플로우)

---

## 프롬프트

### 7-1. 기능 명세 작성 프롬프트

```
AI서비스 UIUX 설계 중. 7단계 기능설계.

[IA 설계]
{6단계 메뉴별 기능 목록 붙여넣기}

[Pain Point]
{4단계 우선순위 상위 항목 붙여넣기}

기능 명세서 작성:

| 기능ID | 기능명 | 상세설명 | 입력 | 출력 | 우선순위(MVP/후순위) | 관련 Pain Point |

AI 기능은 별도 섹션으로 분리:
- AI 기능명
- AI 모델 유형 (분류/추천/생성/예측)
- 입력 데이터
- 출력 형태
- 정확도/성능 요건
```

### 7-2. MVP 범위 정의 프롬프트

```
기능 목록에서 MVP(최소 기능 제품) 범위 정의:

1. MVP 필수 기능 (출시 필수)
2. MVP 권장 기능 (가능하면 포함)
3. 후순위 기능 (향후 버전)

선정 기준:
- Pain Point 해결 영향도
- 개발 복잡도
- 의존성

표 형식으로.
```

### 7-3. 핵심 기능 워크플로우 설계 프롬프트

```
핵심 기능의 워크플로우를 플로우차트 형식으로 설계해줘.

[핵심 기능 목록]
{7-1 결과 중 MVP 필수 기능}

각 핵심 기능별로 다음 요소를 포함한 플로우차트 작성:

1. 트리거: 기능이 시작되는 조건 (사용자 액션 / 시스템 이벤트 / 시간 조건)
2. 입력: 기능 실행에 필요한 데이터
3. 처리: 시스템이 수행하는 로직 (AI 분석 포함)
4. 분기: 조건에 따른 경로 분할 (성공/실패, 정상/주의/위험 등)
5. 출력: 사용자에게 전달되는 결과 (화면/알림/음성 등)
6. 후속 액션: 다음에 가능한 행동

플로우차트 기호:
- [ ] 시작/종료
- < > 분기/조건
- ─── 흐름 연결

텍스트 기반 플로우차트로 작성.
```

### 7-4. AI 기능 상세 설계 프롬프트

```
AI 기능 상세 설계:

[AI 기능 목록]
{7-1 결과 중 AI 기능 부분}

각 AI 기능별로:
1. 기능 시나리오 (사용자 관점)
2. 입력 데이터 스펙
3. AI 처리 로직 개요
4. 출력 형태 및 UI 표현
5. 예외/오류 처리
6. 설명가능성(XAI) 제공 방안

개발자 관점에서 구현 가능하도록 상세히.
```
