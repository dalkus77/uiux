# 8단계: UI/UX 설계

## UI 설계원칙

| 번호 | 원칙명 | 설명 | 적용 예시 |
|------|--------|------|----------|
| 1 | 간결성 우선 | 복잡한 정보를 간단하고 명확하게 전달 | 대시보드에 핵심 지표만 표시, 카드 형태로 정보 구조화 |
| 2 | 사용자 언어 사용 | 전문 용어 대신 일상어 사용 | "건강 상태 파악" vs "건강 분석", "오늘 할 일" vs "일일 목표" |
| 3 | 즉시 피드백 | 사용자 액션에 즉각적인 반응 제공 | 버튼 클릭 시 로딩 표시, 데이터 입력 시 즉시 저장 확인 |
| 4 | 시각적 계층 구조 | 중요 정보를 시각적으로 강조 | 건강 점수를 크게 표시, 색상으로 상태 구분 |
| 5 | 실행 가능한 액션 | 사용자가 바로 실행할 수 있는 명확한 액션 제시 | "오늘 운동하기" 버튼, "지금 시작하기" CTA |

## 색상 가이드

### 색상 팔레트

| 구분 | 색상명 | HEX | 사용 용도 | 선정 이유 |
|------|--------|-----|----------|----------|
| 주조색 (Primary) | 건강 그린 | #4CAF50 | 헤더, 주요 버튼, 브랜드 요소 | 건강과 성장을 상징, 신뢰감 제공 |
| 보조색 (Secondary) | 따뜻한 오렌지 | #FF9800 | 보조 버튼, 강조 영역 | 동기부여와 에너지를 상징 |
| 강조색 (Accent) | 활기찬 블루 | #2196F3 | CTA, 중요 알림 | 신뢰와 전문성을 상징 |
| 배경색 (Background) | 부드러운 화이트 | #FAFAFA | 전체 배경, 카드 배경 | 깔끔하고 편안한 느낌 |
| 표면색 (Surface) | 순수 화이트 | #FFFFFF | 카드, 모달, 입력필드 | 명확한 구분과 가독성 |
| 텍스트 (Primary) | 진한 회색 | #212121 | 본문, 제목 | 높은 가독성 |
| 텍스트 (Secondary) | 중간 회색 | #757575 | 부가 설명, 비활성 | 계층 구조 표현 |

### 의미 색상 (Semantic Colors)

| 상태 | 색상명 | HEX | 배경색 HEX | 사용 예시 |
|------|--------|-----|-----------|----------|
| 성공 (Success) | 성공 그린 | #4CAF50 | #E8F5E9 | 완료, 정상, 승인 |
| 경고 (Warning) | 경고 오렌지 | #FF9800 | #FFF3E0 | 주의, 확인 필요 |
| 오류 (Error) | 오류 레드 | #F44336 | #FFEBEE | 실패, 위험, 거부 |
| 정보 (Info) | 정보 블루 | #2196F3 | #E3F2FD | 안내, 팁 |

### 접근성 검증

| 조합 | 대비율 | WCAG 등급 | 용도 |
|------|--------|----------|------|
| 텍스트 / 배경 | 12.6:1 | AAA | 본문 |
| 건강 그린 / 흰색 | 3.1:1 | AA | 버튼 텍스트 |
| 오류 레드 / 배경 | 4.5:1 | AA | 오류 메시지 |

## AI 서비스 특화 UX 원칙

### 신뢰성 구축

| 요소 | 적용 방안 | 적용 화면 |
|------|----------|----------|
| 투명성 | AI 분석 과정 표시, 데이터 출처 명시 | 건강 분석 결과 화면 |
| 일관성 | 일관된 AI 결과 표시 방식, 예측 가능한 동작 | 모든 AI 기능 화면 |
| 정확성 표시 | 신뢰도 점수, 확률 표시 | 추천 화면, 분석 결과 화면 |

### 설명가능성 (XAI)

| AI 기능 | 설명 제공 방식 | UI 표현 |
|--------|--------------|---------|
| AI-001 (건강 상태 분석) | 각 지표별 기여도, 점수 산출 근거 | "이 점수가 나온 이유" 섹션, 기여도 바 차트 |
| AI-002 (맞춤형 추천) | 추천 이유 텍스트, 매칭 요소 하이라이트 | "왜 추천되었나요?" 버튼, 추천 근거 카드 |
| AI-003 (추천 근거 설명) | 단계별 설명, 사용자 데이터 기반 근거 | 상세 설명 모달, 하이라이트된 데이터 |

### 오류/불확실성 처리

| 상황 | 사용자 메시지 | UI 패턴 |
|------|-------------|---------|
| AI 처리 지연 | "분석 중입니다. 잠시만 기다려주세요" | 로딩 스피너, 진행률 표시 |
| 낮은 신뢰도 | "분석 신뢰도가 낮습니다. 더 많은 데이터를 입력해주세요" | 경고 배지, 신뢰도 바 |
| 처리 실패 | "일시적인 오류가 발생했습니다. 다시 시도해주세요" | 에러 메시지, 재시도 버튼 |
| 결과 없음 | "추천할 정보가 없습니다. 건강 데이터를 더 입력해주세요" | 빈 상태 UI, 안내 메시지 |

### 사용자 제어권

| 제어 요소 | 기능 | UI 컴포넌트 |
|----------|------|-----------|
| AI 결과 수정 | 사용자가 분석 결과에 피드백 제공 | "이 결과가 맞나요?" 피드백 버튼 |
| AI 기능 On/Off | AI 추천 기능 활성화/비활성화 | 설정 화면 토글 스위치 |
| 피드백 제공 | 추천 만족도, 정확성 평가 | 좋아요/별점, 텍스트 피드백 폼 |

## AI UI 패턴 설계

### AI-001: 건강 상태 분석 UI 패턴

**결과 표시:**

| 요소 | 표현 방식 | 비고 |
|------|----------|------|
| 메인 결과 | 큰 숫자로 건강 점수 표시, 색상 코딩 (녹색/노랑/빨강) | 75점, "주의 필요" |
| 신뢰도 | 퍼센트 표시, 신뢰도 바 | "신뢰도: 85%" |
| 근거 | 각 지표별 기여도 바 차트, 텍스트 설명 | "이 점수가 나온 이유" 섹션 |

**UI 목업:**

```
┌────────────────────────────────────┐
│  🤖 AI 건강 분석 결과               │
├────────────────────────────────────┤
│                                    │
│  건강 점수: 75점                    │
│  상태: 주의 필요                    │
│  신뢰도: 85% [신뢰도 바]            │
│                                    │
│  ────────────────────────────────  │
│  이 점수가 나온 이유:               │
│  • 운동 부족: -15점                 │
│  • 식단 불균형: -10점               │
│  • 수면 부족: -5점                  │
│                                    │
│  [상세 분석 보기]                   │
│                                    │
├────────────────────────────────────┤
│  이 분석이 도움이 되었나요?         │
│  [👍 예]  [👎 아니오]               │
└────────────────────────────────────┘
```

**피드백 수집:**

| 피드백 유형 | UI 컴포넌트 | 수집 데이터 |
|-----------|-----------|-----------|
| 만족도 | 좋아요/싫어요 버튼 | boolean |
| 정확성 | 별점 (1-5점) | number |
| 개선 의견 | 텍스트 입력 필드 | text |
