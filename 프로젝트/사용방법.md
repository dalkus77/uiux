# 사용 방법 가이드

## 빠른 시작

### 1단계: 백엔드 서버 시작

프로젝트 루트 폴더에서 `start.bat` 파일을 실행하세요.

```bash
start.bat
```

이 배치 파일이 자동으로:
- ✅ Python 가상환경(.venv) 생성
- ✅ 필요한 패키지 설치 (Flask, flask-cors)
- ✅ 백엔드 서버 시작 (http://localhost:5000)

### 2단계: 프론트엔드 열기

백엔드 서버가 시작된 후:

1. `app/index.html` 파일을 브라우저에서 직접 열기
   - 파일 탐색기에서 더블클릭
   - 또는 브라우저에서 파일 열기

2. 브라우저 주소창에 입력:
   ```
   file:///C:/cursor2/ui/프로젝트/app/index.html
   ```
   (실제 경로에 맞게 수정)

## 중요 사항

### ⚠️ Node.js 불필요
- 이 프로젝트는 **순수 HTML/CSS/JS**를 사용합니다
- Node.js나 npm이 **필요 없습니다**
- 이전 `prototype` 폴더는 더 이상 사용하지 않습니다

### ✅ 필요한 것
- Python 3.12 (C:\python312\python.exe)
- 최신 웹 브라우저 (Chrome, Firefox, Edge 등)

## 문제 해결

### Python 경로 오류
`start.bat` 파일에서 Python 경로를 확인하세요:
- 기본값: `C:\python312\python.exe`
- 다른 경로를 사용하는 경우 `start.bat` 파일 수정

### 포트 충돌
백엔드 서버는 5000 포트를 사용합니다.
다른 프로그램이 5000 포트를 사용 중이면:
- 해당 프로그램 종료
- 또는 `backend/app.py`에서 포트 변경

### CORS 오류
프론트엔드를 `file://` 프로토콜로 열면 CORS 오류가 발생할 수 있습니다.
해결 방법:
1. 간단한 HTTP 서버 사용 (Python 내장 서버)
   ```bash
   cd app
   python -m http.server 8000
   ```
   그 후 브라우저에서 `http://localhost:8000` 접속

2. 또는 백엔드 서버에서 정적 파일 제공하도록 설정

## 프로젝트 구조

```
프로젝트/
├── app/              # 프론트엔드 (순수 HTML/CSS/JS)
│   ├── index.html
│   ├── css/
│   └── js/
├── backend/          # 백엔드 (Python Flask)
│   ├── app.py
│   └── requirements.txt
└── start.bat         # 서버 시작 스크립트
```

## API 엔드포인트

백엔드 서버가 실행되면 다음 API를 사용할 수 있습니다:

- `GET http://localhost:5000/api/health/analysis` - 건강 분석
- `GET http://localhost:5000/api/habits` - 습관 목록
- `POST http://localhost:5000/api/habits` - 습관 추가
- `GET http://localhost:5000/api/recommendations` - 추천 목록
- `POST http://localhost:5000/api/feedback` - 피드백 제출

## 이전 프로토타입 폴더

- `prototype/` - 이전 React 프로토타입 (더 이상 사용 안 함)
- `prototype-simple/` - 간단한 HTML 버전 (참고용)

현재 사용하는 폴더는 `app/`입니다.
